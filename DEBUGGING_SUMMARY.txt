PROJECT: xKG / PaperBench Agent Startup
ISSUE: Agent startup enters infinite loop, never produces agent.log
STATUS: ✅ FIXED and VERIFIED

================================================================================
                              THE PROBLEM
================================================================================

When running:
    ./experiments/paperbench/project/paperbench/scripts/aisi-basic/run.sh

User observed:
    - Repeated log messages: "Attempting to start a cluster instance..."
    - No progress for 10+ minutes
    - Eventually timeout after 600 seconds
    - No agent.log file ever created
    - Dead loop with no error messages

ROOT CAUSE:
    The run.sh script computed PAPERBENCH_DIR but never changed to it before
    running 'docker build'. This caused the COPY command in the Dockerfile to
    look for agent files in the wrong directory, resulting in either:
    - Silent build failure, producing image without agent code
    - Build hanging indefinitely

    Without agent code in the container, execution failed but with no clear
    error message, leading to the apparent "dead loop" (infinite retries).

================================================================================
                              THE SOLUTION
================================================================================

FILE CHANGED:
    experiments/paperbench/project/paperbench/scripts/aisi-basic/run.sh

CHANGE:
    Added: cd "${PAPERBENCH_DIR}"
    Before: docker build -t "${NEW_IMAGE}" -f - . <<'EOF'

EFFECT:
    ✓ Build context now correctly points to directory with agent files
    ✓ COPY command can find ./paperbench/agents/aisi-basic-agent/
    ✓ Docker image builds successfully with agent code
    ✓ Container starts and executes agent
    ✓ agent.log file is produced

================================================================================
                          TECHNICAL DETAILS
================================================================================

ORIGINAL ISSUE (Why It Happened):
    Script developer likely tested from within paperbench directory, where
    the implicit working directory matched PAPERBENCH_DIR. When called from
    other directories, working directory mismatch broke relative paths.

WHY IT LOOKED LIKE A DEAD LOOP:
    1. Docker build runs from wrong directory
    2. COPY command fails to find files silently
    3. Image builds without agent code
    4. Cluster starts container but no agent to execute
    5. Alcatraz retry logic kicks in
    6. Loop repeats until 600-second timeout

WHY IT WAS HARD TO DEBUG:
    - Error messages sparse/hidden in Docker build output
    - Hung process inside Docker, not main Python
    - System logs don't show Docker build output clearly
    - High-level error "cluster won't start" masks root cause
